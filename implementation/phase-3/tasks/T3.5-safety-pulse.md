# T3.5 — Psychological Safety Pulse

**Phase:** 3  
**Owner:** Pilot-team Engineering Manager  
**Estimate:** 1 day  
**Depends on:** T0.5

## Objective

Re-administer the psychological safety pulse survey and compare against Phase 0 baseline.

## Acceptance criteria

- [ ] Same 6 questions administered anonymously.
- [ ] All team members responded.
- [ ] Average score compared against Phase 0 baseline.
- [ ] Score is >= 3.5. If not, pause experiment and run retro.
- [ ] Results documented in Phase 3 exit report.

## Technical notes

```text
Same 6 questions as Phase 0 (docs/psychological-safety.md):
1. I feel safe declaring AI usage in my work.
2. I feel safe challenging AI-generated output.
3. I feel safe saying I do not understand AI-generated code.
4. Our team reviews AI-assisted code carefully.
5. AI metrics are used to improve the system, not blame individuals.
6. I do not feel pressured to use AI when it is not helpful.

Comparison:
  Phase 0 baseline: e.g. 3.8
  Phase 3 result: e.g. 3.9

  If Phase 3 >= Phase 0: safety maintained or improved. Proceed.
  If Phase 3 < Phase 0 but >= 3.5: safety slightly declined. Investigate but proceed.
  If Phase 3 < 3.5: PAUSE. Run retro. Do not proceed to Phase 4.

Also compare individual question scores to identify specific concerns:
  e.g. "I feel safe challenging AI-generated output" dropped from 4.2 to 3.1
  -> indicates developers feel pressured to accept AI output.
```
