# T3.7 — Developer In-PR View

**Phase:** 3  
**Owner:** Platform Engineer  
**Estimate:** 2 days  
**Depends on:** T3.1, T3.2

## Objective

Ship the developer-facing in-PR view inside GitHub.

## Acceptance criteria

- [ ] Shows: AI-assisted (yes/no), metadata status, risk level, suggested reviewers.
- [ ] Lightweight panel, not a large dashboard.
- [ ] No personal scoring shown.
- [ ] Intended experience: "This tool helps me merge safely."

## Technical notes

```text
The in-PR view is delivered via PR comments (from T3.1) and optionally a GitHub App check run.

Components:
  1. PR Comment (from T3.1): guidance and recommendations.
  2. PR Check Run (optional): a lightweight status check that shows:
     - AI-assisted: yes/no/inferred
     - Risk level: low/medium/high
     - Metadata: complete/incomplete
     - This is informational only, not a required check in Phase 3.

Check run format:
  name: "AI Delivery Status"
  status: completed
  conclusion: success (always — never fails in Phase 3)
  output:
    title: "AI-assisted | Risk: Medium | Metadata: Complete"
    summary: "This PR is AI-assisted. Risk score: 7. Recommended: senior review."
    text: "View full details in the platform dashboard."

  This gives developers a quick visual indicator in the GitHub PR checks section
  without requiring them to leave GitHub.
```

## Test cases

```text
- AI-assisted PR -> check run shows "AI-assisted | Risk: {level}"
- Non-AI PR -> check run shows "Non-AI | Metadata: Complete"
- Unknown metadata -> check run shows "Unknown | Metadata: Incomplete"
- Check run always succeeds (never blocks).
```
