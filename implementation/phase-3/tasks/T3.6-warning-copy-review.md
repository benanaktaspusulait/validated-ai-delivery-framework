# T3.6 — Warning Copy Review

**Phase:** 3  
**Owner:** Tech Lead + Pilot team  
**Estimate:** 1 day  
**Depends on:** T3.1

## Objective

Review all PR comment templates with the pilot team and adjust wording based on feedback.

## Acceptance criteria

- [ ] All comment templates shown to pilot team.
- [ ] Feedback collected: "Does this feel helpful or punitive?"
- [ ] Wording adjusted based on feedback.
- [ ] Updated copy recorded in Phase3 config_changes.yaml.

## Technical notes

```text
Meeting format (30 minutes):
  1. Show each comment template (from T3.1).
  2. For each template, ask:
     - "Does this feel helpful or punitive?"
     - "Is the language clear?"
     - "What would you change?"
  3. Collect feedback on:
     - Tone (authoritative vs advisory)
     - Specificity (too vague vs too prescriptive)
     - Actionability (can you actually do what it suggests?)
  4. Adjust wording based on consensus.
  5. Record final copy in config_changes.yaml.

Example adjustments:
  Before: "This PR is high risk and requires senior review."
  After:  "This PR touches payment domain code. A senior reviewer can help catch edge cases."

  Before: "AI metadata is missing. Add it or the PR will be blocked."
  After:  "No AI usage declared. If AI was used, filling in the template helps the team learn."
```
