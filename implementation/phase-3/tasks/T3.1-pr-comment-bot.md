# T3.1 — PR Comment Bot

**Phase:** 3  
**Owner:** Platform Engineer  
**Estimate:** 3 days  
**Depends on:** T2.6, T2.14

## Objective

Build the PR Comment Bot that posts non-blocking guidance on AI-assisted PRs.

## Acceptance criteria

- [ ] Triggered on pull_request events (opened, edited, synchronize, reopened).
- [ ] Checks for AI metadata completeness.
- [ ] Posts warning for PRs > 300 LOC.
- [ ] Posts risk warning for risk_score >= 10.
- [ ] Posts info for low confidence declarations.
- [ ] Never blocks. Never sets a required check.
- [ ] Comments tracked: pr_bot_comments_total.

## Technical notes

```text
Implementation: GitHub Action or platform webhook consumer.

Trigger:
  on:
    pull_request:
      types: [opened, edited, synchronize, reopened]

Logic (non-blocking — all comments are advisory):
  1. Read PR metadata from the platform API:
     GET /api/v1/pull-requests/{prId}
     -> ai_assisted, ai_assistance_confidence, changed_lines, risk_score

  2. Classification:
     a. ai_assisted = false (explicitly non-AI) -> NO COMMENT. Skip.
     b. ai_assisted IS NULL (unknown) -> INFO comment:
        "This PR has no AI usage metadata. If AI was used, please fill in the PR template."
     c. ai_assisted = true AND changed_lines > 300 -> WARNING comment:
        "This AI-assisted PR has {N} lines changed. Consider splitting into smaller PRs
         for faster review and lower risk."
     d. ai_assisted = true AND risk_score >= 10 -> WARNING comment:
        "This AI-assisted PR is high risk (score {N}).
         Signals: {signals list}.
         Recommended reviewers: {required_reviewers}."
     e. ai_assisted = true AND ai_assistance_confidence = "low" -> INFO comment:
        "Low AI confidence declared. Please ensure you understand the AI-generated code,
         have added tests, and can explain this change in production."
     f. Multiple conditions: post the highest-severity comment only.

  3. Comment format (GitHub Markdown):
     ---
     ## 🤖 AI Delivery Guidance
     **Status:** {⚠️ Warning | ℹ️ Info}
     **Reason:** {specific reason}
     **Recommended actions:**
     - {action 1}
     - {action 2}
     [View in dashboard]({link_to_pr_risk_view})
     ---
     *This is guidance, not enforcement. The platform does not track individuals.*

  4. Deduplication: before posting, check if a bot comment already exists on this PR.
     Use GitHub API: GET /repos/{owner}/{repo}/issues/{pr_number}/comments
     If a comment from the bot user already exists with the same type: update it, don't post a new one.

  5. NEVER:
     - Set a check status (no github.status.create).
     - Block the merge.
     - Use threatening or punitive language.

  6. Track observability:
     pr_bot_comments_total{recommendation_type="large_pr", team="payments-team"} += 1
     time_to_comment = now() - webhook_received_at
```

## Test cases

```text
Unit tests:
  - PR with ai_assisted=true, changed_lines=400 -> warning about splitting.
  - PR with ai_assisted=true, risk_score=12 -> warning about reviewers.
  - PR with ai_assisted=true, confidence="low" -> info about understanding code.
  - PR with ai_assisted=false -> no comment.
  - PR with ai_assisted=NULL -> info about missing metadata.
  - PR with multiple conditions -> highest severity comment only.
  - Duplicate comment detection -> update, not create.

Integration test:
  - Open a test PR -> comment appears within 60 seconds.
  - Edit the PR -> comment updates (not duplicated).
  - PR with emergency-fix label -> no comment.
```
