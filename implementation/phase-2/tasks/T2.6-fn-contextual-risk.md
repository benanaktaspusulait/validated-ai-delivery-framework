# T2.6 — fn_contextual_risk

**Phase:** 2  
**Owner:** Platform Engineer  
**Estimate:** 2 days  
**Depends on:** T1.5

## Objective

Implement the AI Contextual Risk Score function.

## Acceptance criteria

- [ ] Reads 10 risk factors from PR and repository data.
- [ ] Applies weights summing to 1.0.
- [ ] Returns risk_score (0-15 scale) and signals list.
- [ ] Suggests required reviewers based on score.

## Technical notes

```text
Weights: W_task=0.15, W_exp=0.05, W_domain=0.15, W_size=0.15, W_security=0.15,
         W_cross=0.10, W_reviewer=0.05, W_familiarity=0.10, W_freshness=0.05, W_boundary=0.05

Each factor scored per docs/risk-policy-engine.md tables:
  Task type: Documentation=0, Test gen=0-1, Internal tooling=0-2, Business logic=1-3, Payment/auth=2-5
  Developer experience: Senior=0, Mid=1, Junior=2
  Domain criticality: Low=0, Medium=1, High=3
  Changed lines: <50=0, 50-150=1, 150-300=2, >300=3
  Security sensitivity: None=0, Infra=1, Auth/payment=3
  Cross-team: None=0, Shared lib=1, Contract=2
  Reviewer load: Light=0, Normal=1, Heavy=2
  Codebase familiarity: >20 commits=0, 5-20=1, <5=2
  Change freshness: Stable=0, 30d=1, High churn=2
  Ownership boundary: Same team=0, Shared=1, Cross-team=2

Final score = SUM(factor_score * weight) * 15 (to get 0-15 scale)

Signals: list of factors with score >= 2.
Required reviewers: based on risk_score thresholds.
```

## Test cases

```text
Fixture: all low-risk factors -> score near 0
Fixture: payment domain + 400 LOC + junior dev + high churn -> score near 12-15
```
