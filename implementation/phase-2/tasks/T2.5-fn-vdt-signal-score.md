## Objective

Implement the Validated Delivery Trend (VDT) Signal Score.

## Acceptance criteria

- [ ] Computes lead time difference and defect difference between AI and non-AI cohorts.
- [ ] Normalises by non-AI standard deviation.
- [ ] Returns signal score, confidence interval (95% CI).
- [ ] Per-sprint granularity, 90-day rolling window.
- [ ] Displayed as trend chart only, never a single KPI number.
- [ ] Mandatory disclaimer included in UI.

## Technical notes

```text
VDT is NOT a single number. It is a 90-day trend chart (per-sprint data points).

Calculation per sprint:
  lt_diff = mean(non_ai_lead_time) - mean(ai_lead_time)
  defect_diff = mean(non_ai_defect_rate) - mean(ai_defect_rate)
  signal = (lt_diff / stdev(non_ai_lead_time)) + (defect_diff / stdev(non_ai_defect_rate))

Confidence interval:
  Using bootstrap or t-distribution approximation.
  If n < 20 per cohort: widen neutral band to [-1.0, 1.0] and annotate "low sample size".

Display:
  Chart type: line chart, x-axis = sprint, y-axis = signal score.
  Reference lines at +0.5 and -0.5 (positive/neutral/negative zones).
  Disclaimer text: "This chart shows a correlational trend. It cannot be used as a causal ROI calculation."
```
