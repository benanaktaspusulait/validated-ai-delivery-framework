# T1.10 — Collector Tests

**Phase:** 1  
**Owner:** Platform Engineer  
**Estimate:** 2 days  
**Depends on:** T1.1, T1.2, T1.3, T1.4, T1.5

## Objective

Write comprehensive tests for all collector components: unit tests, integration tests, contract tests, and load tests.

## Acceptance criteria

- [ ] Unit tests for GitHub payload parser (with recorded fixtures).
- [ ] Unit tests for Jira payload parser (with recorded fixtures).
- [ ] Integration test: webhook receipt -> raw_events write -> normalized table update.
- [ ] Regression test: webhook replay idempotency (duplicate event -> no duplicate row).
- [ ] Contract test: collector against recorded provider payloads.
- [ ] Load test: sustained 100 webhooks/second for 10 minutes.
- [ ] All tests pass.

## Technical notes

```text
Test fixtures:
  test/fixtures/github/
    pull_request_opened.json
    pull_request_merged.json
    pull_request_review_submitted.json
    pull_request_comment.json
  test/fixtures/jira/
    issue_created.json
    issue_updated.json
    sprint_started.json

Unit tests:
  - Parse each fixture -> verify extracted fields.
  - Parse malformed payload -> verify error handling.
  - Verify idempotency key generation.

Integration tests:
  - Send webhook -> verify row in raw_events.
  - Process raw_event -> verify row in pull_requests.
  - Send duplicate webhook -> verify no duplicate row.

Contract tests:
  - For each fixture: verify parser handles all required fields.
  - If a field is missing in the fixture: verify graceful handling (null default).

Load test:
  - Send 100 webhooks/second for 10 minutes (60,000 total).
  - Verify: all stored, no data loss, p99 latency < 5 seconds.
```

## Validation

```text
1. All unit tests pass.
2. All integration tests pass.
3. Contract tests pass with all fixtures.
4. Load test: 60,000 events processed, 0 failures, p99 < 5s.
```
