# T1.6 — Data Confidence Score Job

**Phase:** 1  
**Owner:** Platform Engineer  
**Estimate:** 2 days  
**Depends on:** T1.5

## Objective

Build a batch job that computes the Data Confidence Score for each team and metric.

## Acceptance criteria

- [ ] Job runs nightly (or on demand).
- [ ] Computes confidence score for each team using the 4-component model: volume, freshness, completeness, stability.
- [ ] Piecewise freshness decay applied (0-6h: 100, 6-12h: 75, 12-24h: 50, 24-48h: 25, 48h+: 0).
- [ ] Minimum sample size check for stability (5 data points).
- [ ] Results stored in `metric_snapshots` with data_confidence_score and data_confidence label.
- [ ] confidence_issue populated when score is low or degraded.
- [ ] Unit tests pass with known-answer fixtures.

## Technical notes

```text
Job trigger: cron (every night at 02:00 UTC) or manual POST /api/v1/admin/run-confidence-job.

For each team, for each metric:
1. Volume: COUNT relevant events in last 30 days.
   score = min(100, (count / 100) * 100)

2. Freshness: minutes since most recent event for this team.
   Piecewise:
     0-360 min (0-6h):   score = 100
     360-720 min (6-12h): score = 75
     720-1440 min (12-24h): score = 50
     1440-2880 min (24-48h): score = 25
     2880+ min (48h+):    score = 0

3. Completeness: share of required fields populated.
   Required fields on pull_requests: author_id, repository_id, source_timestamp, ai_assisted IS NOT NULL
   score = (populated / expected) * 100

4. Stability: coefficient of variation (CV) over last 7 days.
   Requires minimum 5 data points. Below 5: score = 30, confidence_issue = "insufficient_sample_size".
   CV < 10%: score = 100
   CV 10-30%: score = 75
   CV 30-50%: score = 50
   CV > 50%: score = 20

5. Final score:
   confidence_score = (volume * 0.30) + (freshness * 0.25) + (completeness * 0.25) + (stability * 0.20)
   Round to integer.

6. Label:
   >= 90: "high"
   70-89: "medium"
   < 70: "low"

7. Store:
   INSERT INTO metric_snapshots (team_id, metric_name, data_confidence_score, data_confidence, confidence_issue, calculation_version, created_at)
   VALUES (team_id, 'confidence_{metric}', score, label, issue, 'v1', now())
```

## Validation

```text
1. Run with fixture data: 80 events, 60min freshness, 19/20 fields, CV 12% -> score ~88 (medium).
2. Run with stale data: 48h+ no events -> freshness = 0, score drops.
3. Run with thin sample: 2 data points -> stability = 30, confidence_issue set.
4. Re-run -> scores are deterministic for same input.
```
