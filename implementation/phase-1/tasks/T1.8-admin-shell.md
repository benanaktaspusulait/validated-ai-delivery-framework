# T1.8 — Admin Shell (SSO + Integration Connect)

**Phase:** 1  
**Owner:** Platform Engineer  
**Estimate:** 3 days  
**Depends on:** T0.1, T0.2

## Objective

Build the admin login page (SSO) and the integration connect/team-mapping screen.

## Acceptance criteria

- [ ] SSO login page redirects to Keycloak (or your SSO provider).
- [ ] After login, admin sees the integration connect screen.
- [ ] GitHub org can be selected from a dropdown (fetched from GitHub App installations).
- [ ] Jira project key can be entered and validated.
- [ ] Team-to-repo mapping can be configured (team name -> repos -> Jira project).
- [ ] Configuration stored in teams and repositories tables.
- [ ] Observation Mode is the default for all new teams.

## Technical notes

```text
Pages:
1. /login -> redirect to Keycloak OAuth2 flow.
2. /admin/integrations -> integration connect screen.
3. /admin/teams -> team-to-repo mapping screen.

Integration connect screen:
  - GitHub section:
    * Show installed GitHub App orgs (GET /app/installations via GitHub API).
    * Select org -> list repos.
    * Confirm installation.
  - Jira section:
    * Enter Jira base URL and API token.
    * Click "Test Connection" -> validates token + shows project list.
    * Select project.

Team mapping screen:
  - Table: Team Name | Repositories | Jira Project | Domain Criticality | Seniority Mix
  - Add team: form with team name, select repos from GitHub, enter Jira key, select criticality, enter senior/mid/junior counts.
  - Save -> INSERT INTO teams + INSERT INTO repositories.
  - Default mode: Observation Mode (no enforcement).
```

## Validation

```text
1. Login via SSO -> redirected to admin dashboard.
2. Connect GitHub org -> repos listed.
3. Connect Jira project -> validated.
4. Create team mapping -> team appears in teams table.
5. New team starts in Observation Mode.
```
