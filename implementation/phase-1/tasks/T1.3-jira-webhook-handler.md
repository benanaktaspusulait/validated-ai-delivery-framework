# T1.3 — Jira Webhook Handler

**Phase:** 1  
**Owner:** Platform Engineer  
**Estimate:** 2 days  
**Depends on:** T0.2

## Objective

Implement `POST /api/v1/webhooks/jira` that receives Jira webhook events, validates tokens, and stores raw payloads.

## Acceptance criteria

- [ ] Endpoint accepts POST requests with Jira webhook payloads.
- [ ] Atlassian Connect token validated.
- [ ] Invalid tokens return 401.
- [ ] Event type extracted from the webhook payload.
- [ ] Raw payload stored in `raw_events` table.
- [ ] Returns 202 Accepted on success.
- [ ] Unit tests pass with recorded Jira webhook fixtures.

## Technical notes

```text
Endpoint: POST /api/v1/webhooks/jira

Headers to validate:
  Atlassian-Connect-Token: {token}

Processing flow:
1. Validate Atlassian-Connect-Token header.
2. Parse webhook payload (Jira sends event type in the payload body, not a header).
3. Event types:
   - jira:issue_created
   - jira:issue_updated
   - jira:issue_deleted
   - sprint_started
   - sprint_completed
4. Generate idempotency_key = sha256(body + issue_key + timestamp).
5. INSERT INTO raw_events with provider = 'jira'.
6. Return 202 Accepted.

Jira webhook setup:
  In Jira > System > WebHooks > Create a WebHook:
  URL: https://your-platform.com/api/v1/webhooks/jira
  Events: Issue created, Issue updated, Sprint started, Sprint completed
  JQL filter: project = {pilot_project_key}
```

## Validation

```text
1. Send a valid Jira webhook payload -> 202, row in raw_events.
2. Replay -> 202, no duplicate.
3. Invalid token -> 401.
4. Unit tests pass with fixtures in test/fixtures/jira/.
```
