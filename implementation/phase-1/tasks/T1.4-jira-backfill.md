# T1.4 — Jira Backfill Script

**Phase:** 1  
**Owner:** Platform Engineer  
**Estimate:** 1 day  
**Depends on:** T1.3

## Objective

Build a script that fetches the last 30 days of issues from the pilot Jira project and stores them as raw events.

## Acceptance criteria

- [ ] Script fetches all issues from the pilot project for the last 30 days.
- [ ] Each issue stored as a raw_event with event_type = "backfill_jira_issue".
- [ ] Issue links (parent, blocked by, etc.) captured.
- [ ] Sprint and team metadata captured.
- [ ] Pagination handled (Jira returns max 100 per page).
- [ ] Idempotent.

## Technical notes

```text
Jira REST API:
  GET /rest/api/3/search?jql=project={key}&maxResults=100&startAt={offset}

  Headers:
    Authorization: Basic {base64(email:api_token)}
    Accept: application/json

  Response: { issues: [...], total: N, startAt: N, maxResults: 100 }

For each issue:
1. Store as raw_event:
   event_type = "backfill_jira_issue"
   external_id = issue key (e.g. PILOT-123)
   idempotency_key = "jira:backfill:{project}:{issue_key}"
   source_timestamp = issue created date
   payload = full issue JSON (includes fields, links, sprint, assignee)

2. Extract issue links:
   GET /rest/api/3/issue/{issue_key}?fields=issuelinks
   Store linked issues in payload.

3. Extract sprint metadata:
   Sprint info is in the issue's custom field (varies by Jira config).
   Parse sprint name, start date, end date, state.
```

## Validation

```text
1. Run against pilot project -> raw_events populated.
2. Re-run -> no duplicates.
3. Issue links captured correctly.
```
