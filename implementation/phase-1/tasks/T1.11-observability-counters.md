# T1.11 — Observability Counters

**Phase:** 1  
**Owner:** Platform Engineer  
**Estimate:** 1 day  
**Depends on:** T1.1, T1.5

## Objective

Instrument all collector components with observability metrics (counters, histograms, gauges).

## Acceptance criteria

- [ ] `collector_events_received_total` counter (by provider, event_type).
- [ ] `collector_processing_failures_total` counter (by provider, error_class).
- [ ] `raw_events_dead_letter_total` counter.
- [ ] `retry_attempts_total` counter.
- [ ] `source_reconciliation_gap_percent` gauge (nightly).
- [ ] `webhook_lag_seconds` histogram (source_timestamp to received_at).
- [ ] `ingestion_throughput_per_second` gauge.
- [ ] Alerts configured: gap > 5%, lag > 1 hour.

## Technical notes

```text
Metrics format: Prometheus (or your observability stack).

Counters:
  collector_events_received_total{provider="github", event_type="pull_request"} += 1
  collector_processing_failures_total{provider="github", error_class="rate_limit"} += 1
  raw_events_dead_letter_total += 1
  retry_attempts_total += 1

Gauges:
  source_reconciliation_gap_percent = 3.2
  ingestion_throughput_per_second = 85.0

Histograms:
  webhook_lag_seconds = 2.3 (observe on each webhook receipt)

Alerts (Prometheus alerting rules or equivalent):
  - alert: HighReconciliationGap
    expr: source_reconciliation_gap_percent > 5
    for: 1h
    annotations: { summary: "Reconciliation gap exceeds 5%" }

  - alert: HighWebhookLag
    expr: histogram_quantile(0.99, webhook_lag_seconds) > 3600
    for: 15m
    annotations: { summary: "Webhook p99 lag exceeds 1 hour" }
```

## Validation

```text
1. Send 10 webhooks -> collector_events_received_total incremented by 10.
2. Send invalid webhook -> collector_processing_failures_total incremented.
3. Reconciliation job runs -> source_reconciliation_gap_percent updated.
4. Alerts fire when thresholds exceeded.
```
