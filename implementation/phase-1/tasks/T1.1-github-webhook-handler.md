# T1.1 — GitHub Webhook Handler

**Phase:** 1  
**Owner:** Platform Engineer  
**Estimate:** 3 days  
**Depends on:** T0.1

## Objective

Implement `POST /api/v1/webhooks/github` that receives GitHub webhook events, validates signatures, and stores raw payloads.

## Acceptance criteria

- [ ] Endpoint accepts POST requests with GitHub webhook payloads.
- [ ] HMAC-SHA256 signature validated (X-Hub-Signature-256 header).
- [ ] Invalid/missing signatures return 401 Unauthorized.
- [ ] Event type extracted from X-GitHub-Event header.
- [ ] Raw payload stored in `raw_events` table with correct idempotency key.
- [ ] Returns 202 Accepted on success.
- [ ] Handles rate limits with exponential backoff (1s, 2s, 4s, 8s, max 30s).
- [ ] Unit tests pass with recorded GitHub webhook fixtures.

## Technical notes

```text
Endpoint: POST /api/v1/webhooks/github

Headers to validate:
  X-Hub-Signature-256: sha256={HMAC-SHA256 of body using webhook secret}
  X-GitHub-Event: event type (pull_request, pull_request_review, etc.)
  X-GitHub-Delivery: unique delivery ID (use as dedup key if idempotency_key is null)

Processing flow:
1. Read raw body bytes.
2. Compute HMAC-SHA256(body, webhook_secret).
3. Compare with X-Hub-Signature-256 header. If mismatch -> 401.
4. Parse X-GitHub-Event header to determine event type.
5. Generate idempotency_key = sha256(body + event_type + timestamp).
6. INSERT INTO raw_events (id, provider, event_type, external_id, idempotency_key, source_timestamp, payload, processing_status)
   VALUES (uuid(), 'github', event_type, delivery_id, idempotency_key, now(), body::jsonb, 'pending')
   ON CONFLICT (idempotency_key) DO NOTHING (if idempotency_key is not null).
7. Return 202 Accepted with { status: "accepted", event_id: delivery_id }

Event types to handle (store as raw, normalize in ETL):
  pull_request          - PR opened, edited, closed, merged, synchronized
  pull_request_review   - review submitted, edited, dismissed
  pull_request_review_comment - comment on a review thread

Idempotency:
  Use provider + event_type + external_id + source_timestamp as the composite key.
  The unique index on raw_events(idempotency_key) prevents duplicates.
```

## Validation

```text
1. Send a valid GitHub webhook payload -> 202, row in raw_events.
2. Replay the same payload -> 202, no duplicate row.
3. Send with invalid signature -> 401, no row in raw_events.
4. Send with missing X-GitHub-Event -> 400 or appropriate error.
5. Unit tests pass with fixtures in test/fixtures/github/.
```
