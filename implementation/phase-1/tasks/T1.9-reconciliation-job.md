# T1.9 — Nightly Reconciliation Job

**Phase:** 1  
**Owner:** Platform Engineer  
**Estimate:** 1 day  
**Depends on:** T1.1, T1.3, T1.5

## Objective

Build a nightly job that compares provider source counts against local counts to detect data gaps.

## Acceptance criteria

- [ ] Job runs nightly after Data Confidence Score job.
- [ ] Compares GitHub PR count (via API) vs local pull_requests count for last 30 days.
- [ ] Compares Jira issue count (via API) vs local jira_issues count for last 30 days.
- [ ] Logs discrepancy as source_reconciliation_gap_percent.
- [ ] Alerts when gap exceeds 5%.
- [ ] Results stored for the exit report.

## Technical notes

```text
Job trigger: nightly at 03:00 UTC (after confidence job).

For each team's repos:
1. GitHub:
   GET /repos/{owner}/{repo}/pulls?state=all&since={30_days_ago} -> total from API
   SELECT COUNT(*) FROM pull_requests WHERE repository_id = {repo_id} AND created_at_source >= {30_days_ago}
   gap = abs(api_count - local_count) / api_count * 100

2. Jira:
   GET /rest/api/3/search?jql=project={key}&maxResults=0 -> total from API
   SELECT COUNT(*) FROM jira_issues WHERE created_at_source >= {30_days_ago}
   gap = abs(api_count - local_count) / api_count * 100

3. Log: source_reconciliation_gap_percent = max(gap_github, gap_jira)

4. Alert: if gap > 5%, notify Platform Lead via Slack.
```

## Validation

```text
1. Run against pilot data -> gap < 5%.
2. Simulate missing data (delete some records) -> gap increases.
3. Gap > 5% -> alert fires.
```
