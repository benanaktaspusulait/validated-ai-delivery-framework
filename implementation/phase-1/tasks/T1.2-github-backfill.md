# T1.2 — GitHub Backfill Script

**Phase:** 1  
**Owner:** Platform Engineer  
**Estimate:** 2 days  
**Depends on:** T1.1

## Objective

Build a script that fetches the last 30 days of PRs from pilot repositories via the GitHub REST API and stores them as raw events.

## Acceptance criteria

- [ ] Script fetches all PRs (open, closed, merged) from pilot repos for the last 30 days.
- [ ] Each PR stored as a raw_event with event_type = "backfill_pull_request".
- [ ] Pagination handled correctly (GitHub returns max 100 per page).
- [ ] Rate limits respected (403 with X-RateLimit-Remaining -> backoff).
- [ ] Idempotent: re-running does not create duplicates.
- [ ] Progress logged (PRs fetched / total estimated).

## Technical notes

```text
GitHub REST API:
  GET /repos/{owner}/{repo}/pulls?state=all&since={30_days_ago}&per_page=100&page={n}

  Headers:
    Authorization: Bearer {github_app_installation_token}
    Accept: application/vnd.github+json

  Response: array of PR objects with:
    - number, title, user.login, created_at, updated_at, merged_at, closed_at
    - changed_files, additions, deletions
    - labels (check for AI-related labels)
    - body (parse for AI metadata checkboxes)

For each PR:
1. Store as raw_event:
   event_type = "backfill_pull_request"
   external_id = PR number
   idempotency_key = "github:backfill:{repo}:{pr_number}"
   source_timestamp = PR created_at
   payload = full PR JSON

2. Also fetch PR reviews:
   GET /repos/{owner}/{repo}/pulls/{pr_number}/reviews
   Store each as raw_event with event_type = "backfill_pull_request_review"

Pagination:
  Link header contains next page URL.
  Continue until no more pages or rate limit reached.

Progress tracking:
  Log: "Fetched {fetched}/{total} PRs from {owner}/{repo}"
```

## Validation

```text
1. Run against pilot repos -> raw_events populated with 30 days of data.
2. Re-run -> no duplicates (idempotency check).
3. Rate limit handling: if 403 received, backs off and retries.
4. All PRs have title, author, files, timestamps in payload.
```
