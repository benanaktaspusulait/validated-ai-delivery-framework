# T0.4 — Branch Protection Registration

**Phase:** 0  
**Owner:** Platform Engineer  
**Estimate:** 1 hour  
**Depends on:** T0.1

## Objective

Register (but NOT activate) a branch protection rule for the main branch in pilot repositories. This ensures the rule exists for Phase 4 activation.

## Acceptance criteria

- [ ] Branch protection rule added for `main` in each pilot repo.
- [ ] Rule includes "Require status checks to pass" option (NOT enabled yet).
- [ ] Rule includes "Require pull request reviews before merging" option (NOT enabled yet).
- [ ] Rule is saved but inactive — PRs can still merge without checks.
- [ ] Documented in [Phase0]_config_changes.yaml.

## Technical notes

```text
1. In each pilot repo > Settings > Branches > Add branch protection rule.
2. Branch name pattern: main
3. Configure (but do NOT enable):
   - [ ] Require status checks to pass before merging
   - [ ] Require pull request reviews before merging
   - [ ] Require conversation resolution before merging
4. Save the rule. It exists but has no active requirements.
5. In Phase 4, you will enable these options and add the AI Metadata Enforcement check.

Document in config_changes.yaml:
  branch_protection:
    repos:
      - owner/example-org
        repo: payments-api
        rule: main
        status: registered_not_activated
        enforcement_phase: 4
```

## Validation

```text
1. Branch protection rule exists in repo Settings > Branches.
2. A PR can still be merged without any checks passing.
3. The rule is documented in config_changes.yaml.
```
