# T0.3 — PR Template Deployment

**Phase:** 0  
**Owner:** Platform Engineer  
**Estimate:** 1 hour  
**Depends on:** None

## Objective

Deploy the AI usage PR template to all pilot repositories so developers see it when creating PRs.

## Step-by-step

```text
Step 1: Get the template
  Read examples/pr-template.md from this repository.
  It contains:
    - AI Assistance checkboxes (6 options)
    - AI Assistance Confidence (Low/Medium/High)
    - Ownership Confirmation (5 checkboxes)
    - Psychological safety note

Step 2: Deploy to each pilot repo
  For each pilot repository:

  Option A: Manual (small number of repos)
    1. Create a new branch: git checkout -b docs/add-pr-template
    2. Create directory: mkdir -p .github
    3. Copy the template content to .github/pull_request_template.md
    4. Commit and push:
       git add .github/pull_request_template.md
       git commit -m "docs: add AI usage PR template"
       git push origin docs/add-pr-template
    5. Create a PR and merge it.

  Option B: Script (multiple repos)
    Write a shell script:
      REPOS=("owner/repo1" "owner/repo2" "owner/repo3")

      for REPO in "${REPOS[@]}"; do
        echo "Deploying to $REPO..."
        # Clone the repo
        git clone git@github.com:$REPO.git /tmp/$REPO
        cd /tmp/$REPO

        # Create the template
        mkdir -p .github
        cp /path/to/pr-template.md .github/pull_request_template.md

        # Commit and push
        git checkout -b docs/add-pr-template
        git add .github/pull_request_template.md
        git commit -m "docs: add AI usage PR template"
        git push origin docs/add-pr-template

        # Create PR via GitHub API
        curl -X POST \
          -H "Authorization: token $GITHUB_TOKEN" \
          -H "Accept: application/vnd.github+json" \
          https://api.github.com/repos/$REPO/pulls \
          -d '{"title":"docs: add AI usage PR template","head":"docs/add-pr-template","base":"main","body":"Adds the AI usage PR template for the AI Delivery Control Plane pilot."}'

        # Auto-merge (if branch protection allows)
        # Or merge manually

        cd /
        rm -rf /tmp/$REPO
      done

Step 3: Verify the template
  For each pilot repo:
    1. Go to the repo on GitHub.
    2. Click "New pull request".
    3. The PR body should auto-populate with the template.
    4. Verify all sections are present:
       - AI Assistance (6 checkboxes)
       - AI Assistance Confidence (3 checkboxes)
       - Ownership Confirmation (5 checkboxes)
       - Safety note at the bottom

Step 4: Test checkbox rendering
  Create a test PR:
    1. Check "[x] No AI assistance used"
    2. Check "[x] High" for confidence
    3. Check all ownership confirmation boxes
    4. Verify the PR body renders correctly in GitHub Markdown.
    5. Close/delete the test PR.
```

## Acceptance criteria

- [ ] Template copied to `.github/pull_request_template.md` in each pilot repo.
- [ ] Template renders correctly in a test PR.
- [ ] All 6 AI usage checkboxes are present and functional.
- [ ] Confidence level checkboxes (Low/Medium/High) are present.
- [ ] Ownership confirmation checkboxes are present.
- [ ] Psychological safety note is visible at the bottom.
- [ ] Template is on the main branch (not a feature branch).

## Validation

```text
1. Open "New pull request" in each pilot repo.
2. PR body contains the full AI usage template.
3. All checkboxes are clickable and render in GitHub Markdown.
4. Test PR with checkboxes checked -> renders correctly.
```

## Common issues

```text
- Template not showing: file must be at .github/pull_request_template.md (exact path).
- Template shows raw markdown: GitHub may cache. Try hard refresh.
- Branch protection blocks merge: add "docs:" prefix to commit message if conventional commits required.
```
