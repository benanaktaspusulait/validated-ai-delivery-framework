# T0.2 — Jira Service Account Preparation

**Phase:** 0  
**Owner:** Platform Engineer  
**Estimate:** 2 hours  
**Depends on:** None

## Objective

Create a Jira API token and document the pilot team's Jira project configuration.

## Step-by-step

```text
Step 1: Create or identify a service account
  Option A: Create a dedicated Jira service account
    - Go to your org's identity provider (Okta, Azure AD, etc.)
    - Create a user: ai-delivery-bot@yourorg.com
    - Give it Jira access (licensed user or appropriate role)

  Option B: Use an existing account
    - Use a shared team account if acceptable

Step 2: Generate API token
  1. Log in as the service account to https://id.atlassian.com/
  2. Go to Security > API tokens
  3. Click "Create API token"
  4. Name: "ai-delivery-control-plane"
  5. Copy the token immediately. It won't be shown again.
  6. Store in your secrets manager:
     jira_api_token: <token>

  If using environment variables:
    JIRA_BASE_URL=https://yourorg.atlassian.net
    JIRA_PROJECT_KEY=PILOT
    JIRA_API_TOKEN=<token>
    JIRA_EMAIL=ai-delivery-bot@yourorg.com

Step 3: Identify pilot team's Jira project
  1. Go to Jira > Projects
  2. Find the pilot team's project (e.g. "PILOT" or "PAYMENTS")
  3. Note the project key (e.g. "PILOT")
  4. Note the project URL: https://yourorg.atlassian.net/projects/PILOT

Step 4: Test API connectivity
  Run this curl command:
    curl -u "ai-delivery-bot@yourorg.com:<API_TOKEN>" \
      -H "Accept: application/json" \
      "https://yourorg.atlassian.net/rest/api/3/project/PILOT"

  Expected response: JSON with project details (id, key, name, etc.)

  Also test issue search:
    curl -u "ai-delivery-bot@yourorg.com:<API_TOKEN>" \
      -H "Accept: application/json" \
      "https://yourorg.atlassian.net/rest/api/3/search?jql=project=PILOT&maxResults=1"

  Expected response: JSON with at least 1 issue.

Step 5: Identify custom fields (if needed)
  Some Jira instances have custom fields for:
  - Sprint (used by Jira Agile)
  - Story points
  - Team assignment

  To find custom fields:
    curl -u "ai-delivery-bot@yourorg.com:<API_TOKEN>" \
      "https://yourorg.atlassian.net/rest/api/3/field"

  Look for fields related to:
  - "Sprint" (usually customfield_10010 or similar)
  - "Story Points" (customfield_10016 or similar)
  - "Team" (customfield_10001 or similar)

  Document these custom field IDs. They'll be needed in T1.4 (Jira backfill).

Step 6: Document in team-config.yaml
  Pre-fill the pilot team config:
    jira:
      base_url: https://yourorg.atlassian.net
      project_key: PILOT
      email: ai-delivery-bot@yourorg.com
      # api_token: stored in secrets manager, NOT in this file
      custom_fields:
        sprint: customfield_10010
        story_points: customfield_10016
```

## Acceptance criteria

- [ ] Jira API token created for the service account.
- [ ] Pilot team's Jira project key identified and documented.
- [ ] Jira base URL documented.
- [ ] API token stored securely (not committed to any repo).
- [ ] Test API call succeeds: `GET /rest/api/3/project/{key}` returns project details.
- [ ] Test issue search succeeds: `GET /rest/api/3/search?jql=project={key}` returns issues.
- [ ] Custom field IDs documented (if applicable).
- [ ] Configuration recorded in team-config.yaml (without the token).

## Validation

```text
1. curl command returns project details (200 OK).
2. curl command returns at least 1 issue.
3. API token works from a clean environment (not cached session).
4. Configuration documented in team-config.yaml.
```

## Common issues

```text
- "Authentication failed": check email + token combination. Token is per-user.
- "Project not found": check project key is uppercase (e.g. "PILOT", not "pilot").
- "Permission denied": service account needs Jira access. Check with Jira admin.
- Custom field IDs vary by Jira instance. Always verify with /rest/api/3/field.
```
