# T0.2 — Jira Service Account Preparation

**Phase:** 0  
**Owner:** Platform Engineer  
**Estimate:** 2 hours  
**Depends on:** None

## Objective

Create a Jira API token and document the pilot team's Jira project configuration.

## Acceptance criteria

- [ ] Jira API token created for the service account.
- [ ] Pilot team's Jira project key identified and documented.
- [ ] Jira base URL documented.
- [ ] API token stored securely (not committed to any repo).
- [ ] Test API call succeeds: `GET /rest/api/3/project/{key}` returns project details.

## Technical notes

```text
1. Create a dedicated Jira service account (or use an existing one).
2. Generate an API token:
   - Atlassian account > Security > API tokens > Create API token.
   - Store the token in your secrets manager.
3. Document in team-config.yaml (pilot team):
   jira:
     base_url: https://yourorg.atlassian.net
     project_key: PILOT
     api_token: (stored in secrets manager, not in file)
4. Test connectivity:
   curl -u service-account@email.com:API_TOKEN \
     https://yourorg.atlassian.net/rest/api/3/project/PILOT
```

## Validation

```text
1. API token authenticates successfully.
2. GET /rest/api/3/project/{key} returns project metadata.
3. GET /rest/api/3/search?jql=project={key}&maxResults=1 returns at least 1 issue.
```
