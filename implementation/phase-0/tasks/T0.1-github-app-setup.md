# T0.1 — GitHub App Setup

**Phase:** 0  
**Owner:** Platform Engineer  
**Estimate:** 2 hours  
**Depends on:** None

## Objective

Create a GitHub App in the pilot organisation that can receive webhook events and read PR data.

## Acceptance criteria

- [ ] GitHub App created in the pilot org (Settings > Developer settings > GitHub Apps).
- [ ] Permissions configured: `pull_requests` (read/write), `pull_request_reviews` (read), `contents` (read), `metadata` (read).
- [ ] Webhook events subscribed: `pull_request`, `pull_request_review`, `pull_request_review_comment`.
- [ ] Private key generated and stored securely (not committed to any repo).
- [ ] App installed on the pilot repositories.
- [ ] Webhook URL documented (will point to the platform's `/api/v1/webhooks/github` endpoint in Phase 1).

## Technical notes

```text
GitHub App creation:
  Name: ai-delivery-control-plane
  Homepage URL: (internal URL or placeholder)
  Webhook URL: (placeholder — will be set in Phase 1)
  Webhook secret: generate a random 40-char hex string, store in secrets manager.

Permissions:
  Repository permissions:
    Pull requests: Read & Write
    Pull request reviews: Read
    Contents: Read
    Metadata: Read (always available)

Events:
  Pull request
  Pull request review
  Pull request review comment

After creation:
  Generate a private key (PEM file).
  Store the private key, app ID, and client secret in your secrets manager.
  Do NOT commit the private key to the repo.
```

## Validation

```text
1. GitHub App appears in the org's installed apps list.
2. Private key can be used to generate a JWT (test with GitHub's API).
3. App is visible in pilot repo Settings > Integrations > GitHub Apps.
```
