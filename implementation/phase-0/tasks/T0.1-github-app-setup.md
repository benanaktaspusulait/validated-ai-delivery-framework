# T0.1 — GitHub App Setup

**Phase:** 0  
**Owner:** Platform Engineer  
**Estimate:** 2 hours  
**Depends on:** None

## Objective

Create a GitHub App in the pilot organisation that can receive webhook events and read PR data. This App is the foundation for all GitHub integration in Phase 1+.

## Step-by-step

```text
Step 1: Create the GitHub App
  1. Go to https://github.com/organizations/{org}/settings/developer-settings
  2. Click "GitHub Apps" > "New GitHub App"
  3. Fill in:
     - GitHub App name: ai-delivery-control-plane
       (must be unique within the org)
     - Homepage URL: https://internal-platform.example.com
       (placeholder — will be replaced in Phase 1)
     - Webhook URL: https://internal-platform.example.com/api/v1/webhooks/github
       (placeholder — will go live in Phase 1)
     - Webhook secret: click "Generate a random secret"
       Copy this value immediately. Store in your secrets manager.
       Format: 40-char hex string (e.g. "a1b2c3d4e5f6...")

Step 2: Set permissions
  Repository permissions:
    Pull requests:        Read & Write
    Pull request reviews: Read
    Contents:             Read
    Metadata:             Read (always available, cannot be changed)

  Organization permissions:
    Members:              Read (to map users to teams)

Step 3: Subscribe to events
  Check these event types:
    [x] Pull request
    [x] Pull request review
    [x] Pull request review comment

Step 4: Create the App
  Click "Create GitHub App"
  Note the following values:
    - App ID: (shown on the App settings page)
    - Client ID: (shown on the App settings page)
    - Client secret: (shown only once — copy immediately)
    - Private key: click "Generate a private key"
      Downloads a .pem file. Store securely.

Step 5: Store secrets
  Add to your secrets manager (HashiCorp Vault, AWS Secrets Manager, etc.):
    github_app_id: <App ID>
    github_client_id: <Client ID>
    github_client_secret: <Client Secret>
    github_webhook_secret: <Webhook Secret>
    github_private_key: <contents of .pem file>

  If using environment variables for pilot:
    GITHUB_APP_ID=<App ID>
    GITHUB_WEBHOOK_SECRET=<Webhook Secret>
    GITHUB_PRIVATE_KEY_PATH=/secrets/github-private-key.pem

Step 6: Install the App on pilot repositories
  1. Go to the App settings page > "Install App"
  2. Select the pilot organisation
  3. Select "Only select repositories"
  4. Choose the pilot team's repositories
  5. Click "Install"
  6. Note the Installation ID (from the URL after install):
     https://github.com/organizations/{org}/settings/installations/{INSTALLATION_ID}

Step 7: Verify the App
  1. Go to the pilot repo > Settings > Integrations > GitHub Apps
  2. Confirm "ai-delivery-control-plane" appears
  3. Confirm it has the correct permissions
  4. Confirm it's subscribed to the correct events

Step 8: Test JWT generation (optional but recommended)
  Write a small script to verify the private key works:
    import jwt, time
    payload = {
      "iat": int(time.time()),
      "exp": int(time.time()) + 600,
      "iss": "<APP_ID>"
    }
    token = jwt.encode(payload, private_key, algorithm="RS256")
    # Use token to call GitHub API:
    # GET /app/installations
    # Should return the installation we just created.
```

## Acceptance criteria

- [ ] GitHub App created with correct name "ai-delivery-control-plane".
- [ ] Permissions: pull_requests (R/W), pull_request_reviews (R), contents (R), metadata (R).
- [ ] Events subscribed: pull_request, pull_request_review, pull_request_review_comment.
- [ ] Private key generated and stored securely (not in any repo).
- [ ] Webhook secret generated and stored securely.
- [ ] App installed on pilot repositories.
- [ ] Installation ID documented in Phase 0 exit report.
- [ ] JWT generation test passes (optional but recommended).

## Validation

```text
1. GitHub App appears in org > Developer settings > GitHub Apps.
2. App is visible in pilot repo > Settings > Integrations > GitHub Apps.
3. Private key can generate a valid JWT (test script passes).
4. Installation ID is recorded.
```

## Common issues

```text
- "GitHub App name already taken": try adding a suffix like "-internal" or "-pilot".
- "Webhook URL must be HTTPS": use a real URL or a tunnel (ngrok) for testing.
- Private key download fails: try a different browser.
- App not appearing in repo: check installation scope (all repos vs selected).
```
