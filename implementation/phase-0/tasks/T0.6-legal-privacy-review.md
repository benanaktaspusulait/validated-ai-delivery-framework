# T0.6 — Legal/Privacy Review

**Phase:** 0  
**Owner:** Legal or privacy counsel  
**Estimate:** 1 week  
**Depends on:** None

## Objective

Obtain written legal/privacy approval for collecting and processing developer activity data (PR metadata, review timestamps, Jira issues).

## Acceptance criteria

- [ ] Legal/privacy counsel has reviewed the data scope defined in docs/governance-and-privacy.md.
- [ ] Written approval received and stored.
- [ ] Data retention policy (12 months detailed, 24 months aggregates) is acceptable.
- [ ] Pseudonymisation approach for developer identifiers is approved.
- [ ] No raw prompt storage is confirmed as compliant.
- [ ] Approval documented in [Phase0]_exit_report.pdf.

## Technical notes

```text
Data scope for review:
- PR metadata: title, author (pseudonymised), files changed, lines changed, timestamps
- Review data: reviewer (pseudonymised), timestamps, comments count, approval status
- Jira data: issue key, type, severity, status, linked PR
- AI metadata: usage types checkbox (not free text), confidence level
- Derived metrics: team-level aggregates only

NOT collected:
- Raw prompt content
- Individual performance scores
- IDE telemetry
- Code content (only metadata about code changes)

Retention:
- Detailed data: 12 months, then purged
- Aggregated metrics: 24 months
- Audit logs: 24 months
```
