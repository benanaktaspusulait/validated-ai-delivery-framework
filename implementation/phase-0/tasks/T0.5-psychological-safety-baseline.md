# T0.5 — Psychological Safety Baseline

**Phase:** 0  
**Owner:** Pilot-team Engineering Manager  
**Estimate:** 2h active work; 24h+ wall-clock (survey deadline)  
**Depends on:** T0.8 (briefing must run first — survey is distributed during the briefing)

## Objective

Collect and analyse the psychological safety baseline survey responses. The briefing and survey distribution happen in T0.8; this task owns collection, calculation and analysis.

## Acceptance criteria

- [ ] Survey responses collected (>= 80% participation).
- [ ] Average score calculated: overall + per-question.
- [ ] Score is >= 3.5. If not, see fail action.
- [ ] Results documented in examples/phase0-exit-report-template.md.

## Implementation steps

```text
Step 1: Wait for survey deadline (24h after T0.8 distributes the survey).

Step 2: Collect responses.
  - Export from Google Forms / Typeform.
  - Verify >= 80% participation (follow up with non-respondents if needed).
  - If < 80% after follow-up: note in exit report, proceed with available data.

Step 3: Calculate scores.
  - Per-question average (6 questions, 1-5 scale).
  - Overall average = sum of all responses / total responses.
  - Example: Q1=4.2, Q2=3.8, Q3=3.5, Q4=4.0, Q5=4.1, Q6=3.9 -> Overall = 3.92.

Step 4: Analyse results.
  - If overall >= 3.5: PROCEED. Record the score.
  - If overall < 3.5: PAUSE. Run retro (see fail action below).
  - Analyse individual questions for specific concerns:
    * Q2 low -> developers feel pressured to accept AI code.
    * Q3 low -> developers submitting code they don't understand.
    * Q5 low -> trust issue with management.
    * Q6 low -> adoption pressure too high.

Step 5: Document in exit report.
  - Use examples/phase0-exit-report-template.md.
  - Fill in: survey method, respondent count, per-question scores, overall average, pass/fail.
```

## Fail action

```text
If score < 3.5:
1. Run blameless retro: "What would need to change for you to feel safe participating?"
2. Address top concern (re-communicate no-individual-use, clarify AI is optional, etc.).
3. Re-administer after 1 week.
4. If still < 3.5: do not proceed to Phase 1. Cancel or reduce scope.
```

## Validation

```text
1. Survey responses collected from >= 80% of team.
2. Overall score >= 3.5.
3. Individual question scores documented.
4. Results in exit report (examples/phase0-exit-report-template.md).
```
