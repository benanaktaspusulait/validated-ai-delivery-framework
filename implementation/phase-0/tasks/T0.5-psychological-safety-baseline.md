# T0.5 — Psychological Safety Baseline

**Phase:** 0  
**Owner:** Pilot-team Engineering Manager  
**Estimate:** 2 hours (survey + analysis)  
**Depends on:** None

## Objective

Measure the pilot team's psychological safety baseline using the 6-question pulse survey. The score must be >= 3.5 to proceed to Phase 1.

## Acceptance criteria

- [ ] 6-question survey administered anonymously to all pilot team members.
- [ ] All team members responded (100% participation).
- [ ] Average score calculated and recorded.
- [ ] Score is >= 3.5. If not, see fail action.
- [ ] Results stored in [Phase0]_exit_report.pdf.

## Technical notes

```text
Questions (from docs/psychological-safety.md):
1. I feel safe declaring AI usage in my work.
2. I feel safe challenging AI-generated output.
3. I feel safe saying I do not understand AI-generated code.
4. Our team reviews AI-assisted code carefully.
5. AI metrics are used to improve the system, not blame individuals.
6. I do not feel pressured to use AI when it is not helpful.

Scoring: 1 = strongly disagree, 5 = strongly agree.
Average = sum of all responses / number of responses.

Administer via:
- Google Forms (anonymous)
- Typeform (anonymous)
- Paper survey (if preferred)

Briefing message (30 minutes before survey):
"We are evaluating whether this team is ready to pilot an AI delivery dashboard. 
The dashboard tracks team-level delivery health, never individual performance. 
Your responses to this survey help us understand whether the team feels safe 
participating. There are no right or wrong answers."
```

## Validation

```text
1. Survey link sent to all pilot team members.
2. All responses collected (anonymously).
3. Average score >= 3.5.
4. Score and methodology documented in exit report.
```

## Fail action

```text
If score < 3.5:
1. Run a blameless retro to understand concerns.
2. Address the top concern (e.g. clarify "no individual performance use" more strongly).
3. Re-administer after 1 week.
4. If still < 3.5: do not proceed to Phase 1. Cancel or reduce scope.
```
