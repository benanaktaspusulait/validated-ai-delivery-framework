# T0.8 — Pilot Team Briefing

**Phase:** 0  
**Owner:** Platform Lead  
**Estimate:** 1-1.5h (30 min prep + 30 min briefing, may extend for Q&A)  
**Depends on:** T0.6, T0.7 (legal and HR approval must be obtained first)

## Objective

Brief the pilot team on the platform's purpose, data usage and cultural rules. Distribute the psychological safety survey. This is the team's first exposure to the project.

## Acceptance criteria

- [ ] Briefing held with all pilot team members.
- [ ] Purpose explained: "We are not measuring individual productivity."
- [ ] Non-negotiable rules communicated.
- [ ] PR template demonstrated.
- [ ] Psychological safety survey distributed.
- [ ] Q&A session held.
- [ ] Attendance recorded for exit report.

## Implementation steps

```text
Step 1: Prepare (30 min before).
  - Book 30-minute meeting with full pilot team.
  - Prepare talking points (see agenda below).
  - Have PR template ready to demo.
  - Create survey (use docs/psychological-safety.md questions) and get link ready.

Step 2: Run briefing (30 minutes).
  Agenda:
    1. Purpose (5 min): "We are NOT measuring individual productivity. We ARE validating whether the system can use AI safely."
    2. What it does (5 min): team-level metrics, no personal dashboards, no rankings.
    3. What it does NOT do (5 min): no individual scoring, no raw prompt storage, no IDE monitoring.
    4. Your role (5 min): fill in PR template, read in-PR guidance, give feedback in retro.
    5. The rules (5 min): no ranking, no raw prompts, no hard enforcement before Phase 4.
    6. Demo PR template (2 min): open pilot repo > New pull request > show template.
    7. Q&A (3 min): open floor. Common Q&A:
       "Will this affect my performance review?" -> No. Absolutely not.
       "Do I have to use AI?" -> No. AI usage is optional.
       "What if I don't fill in the template?" -> Phase 3: gentle reminder. Phase 4: required before merge.

Step 3: Distribute survey.
  - Send survey link immediately after briefing (Slack or email).
  - Message: "Please fill in this anonymous survey within 24 hours. No right or wrong answers."
  - Survey link: (create in Google Forms / Typeform using docs/psychological-safety.md questions).

Step 4: Record attendance.
  - Note who attended (for exit report).
  - Note any questions or concerns.
  - Note follow-up actions needed.

Step 5: After T0.5 completes, assemble the exit report.
  - Use examples/phase0-exit-report-template.md.
  - Fill in all sections: Decision, Evidence, Risks, Key decisions, Open items.
  - Get sign-offs from Platform Lead, Legal, HR, Pilot-team EM.
```

## Validation

```text
1. Briefing held with all team members.
2. All key messages delivered.
3. PR template shown and explained.
4. Survey distributed.
5. Questions answered.
6. Attendance and notes in exit report.
```
