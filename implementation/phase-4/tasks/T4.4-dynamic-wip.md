# T4.4 — Dynamic AI WIP Recommendation

**Phase:** 4  
**Owner:** Platform Engineer  
**Estimate:** 2 days  
**Depends on:** T2.1-T2.3

## Objective

Implement Dynamic AI WIP recommendation with high-risk penalty.

## Acceptance criteria

- [ ] Computes base WIP from team seniority mix.
- [ ] Adjusts based on defect rate and review debt deviations.
- [ ] High-risk penalty triggers when both deviations are elevated.
- [ ] Floor at configured minimum (default 2).
- [ ] Published to dashboard and optionally Jira/GitHub Projects.
- [ ] Reason for change shown to team.

## Technical notes

```text
Base formula:
  base = (senior * 2) + (mid * 1) + (junior * 0.5)

Dynamic adjustment:
  defect_ratio = current_ai_defect_rate / comparable_non_ai_defect_rate
  defect_deviation = defect_ratio - 1

  review_debt_ratio = ai_review_debt_age_ratio
  review_deviation = review_debt_ratio - 1

  seniority_ratio = senior / total_engineers

  adjusted = base - (defect_deviation * 0.5) - (review_deviation * 0.3) + (seniority_ratio * 0.2)

  Always round down when quality risk is increasing.

High-risk penalty (triggers when defect_ratio > 1.5 AND review_debt_ratio > 1.2):
  adjusted = base * 0.7 * (1 - min(defect_deviation, 3) / 10)

Floor:
  configured_min = team-config.yaml wip.min_limit (default 2)
  if adjusted < configured_min:
    adjusted = configured_min
    add recommendation: "WIP at configured floor. Investigate quality signals."

Display:
  Dashboard card:
    Dynamic AI WIP: {adjusted} (base: {base})
    Reason: {driving signals}
    Trend: {arrow} (increasing/decreasing from last sprint)

Publishing:
  Optional: POST to Jira or GitHub Projects to update a field or create a task.
  Required: visible on the Team Dashboard.
```
