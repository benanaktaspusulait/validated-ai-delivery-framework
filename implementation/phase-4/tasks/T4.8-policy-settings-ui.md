# T4.8 — Policy Settings UI

**Phase:** 4  
**Owner:** Platform Engineer  
**Estimate:** 2 days  
**Depends on:** T4.1

## Objective

Build the Policy Settings rule-editor screen.

## Acceptance criteria

- [ ] Enable/disable individual policies.
- [ ] Adjust thresholds (e.g. large PR threshold from 300 to 500).
- [ ] View policy evaluation history.
- [ ] View confidence gate status (active/blocked/warned per policy).
- [ ] Changes stored in config_versions with audit trail.

## Technical notes

```text
Screen: /admin/policies

Sections:
  1. Policy List:
     Table: Policy Name | Enabled | Type | Confidence Gate | Status
     - ai_metadata_required: enabled, blocking, >= 90, active
     - large_ai_pr_warning: enabled, warning, >= 70, active
     - high_risk_reviewer: enabled, warning, >= 70, active
     - prompt_leakage_scan: enabled, comment-only, N/A, active

  2. Threshold Editor:
     - Large PR threshold: [300] LOC (input field)
     - High risk score threshold: [10] (input field)
     - Review debt warning threshold: [1.5]x (input field)
     - Emergency override limits: soft [3], hard [5] per sprint

  3. Confidence Gate Status:
     For each policy: which metric backs it, current confidence score, gate status.
     - Green: confidence >= 90 (blocking allowed)
     - Yellow: confidence 70-89 (warn only)
     - Red: confidence < 70 (auto-disabled)

  4. Evaluation History:
     Last 100 policy evaluations: timestamp, policy, PR, action taken, confidence at time.

Storage:
  All changes -> INSERT INTO config_versions (config_type, config_key, config_value, created_by, created_at)
  Audit log -> INSERT INTO audit_log (entity_type='policy', entity_id=policy_name, action='update', old_value, new_value, performed_by)
```

## Test cases

```text
- Toggle policy enabled/disabled -> change persisted, audit logged.
- Change threshold from 300 to 500 -> new threshold applies to next evaluation.
- View confidence gate -> shows current status for each policy.
- View evaluation history -> last 100 entries displayed.
```
