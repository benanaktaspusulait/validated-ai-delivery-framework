# T4.1 — AI Metadata Blocker Activation

**Phase:** 4  
**Owner:** Platform Engineer  
**Estimate:** 2 days  
**Depends on:** T0.4, T3.1

## Objective

Activate the AI Metadata Blocker as a required status check on pilot repositories.

## Acceptance criteria

- [ ] Branch protection rule activated (from T0.4 registration).
- [ ] AI Metadata Enforcement added as required check.
- [ ] PRs without AI metadata cannot merge.
- [ ] Emergency-fix label bypasses the check.
- [ ] Override logged to policy_overrides table.

## Technical notes

```text
Steps:
1. Go to repo Settings > Branches > main branch protection rule.
2. Enable "Require status checks to pass before merging".
3. Add "AI Metadata Enforcement" as a required status check.
4. The workflow (examples/github-actions/ai-metadata-enforcement.yml) now blocks merge.

Workflow flow:
  on: pull_request [opened, edited, synchronize, reopened]

  Step 1: Check for "emergency-fix" label.
    If present: post notice, skip all checks, log override, return.

  Step 2: Parse PR body for AI metadata.
    Check: hasNoAI = body includes "[x] No AI assistance used"
    Check: hasAI = body includes any of the 5 AI usage categories
    Check: hasConfidence = body includes "[x] Low/Medium/High"

  Step 3: Evaluate.
    If !hasNoAI && !hasAI: FAIL("AI metadata required")
    If hasAI && !hasConfidence: FAIL("Confidence required when AI is used")
    If hasNoAI || (hasAI && hasConfidence): PASS

  Step 4: On PASS: update check status to success.
           On FAIL: update check status to failure with descriptive message.

Override logging:
  When emergency-fix label is detected:
    POST /api/v1/policy-overrides
    {
      "pull_request_id": "pr_123",
      "policy_name": "ai_metadata_required",
      "override_label": "emergency-fix",
      "overridden_by": "github_actor_login",
      "reason": "Emergency production fix"
    }

Confidence gate integration:
  Before blocking, check Data Confidence Score >= 90 for the backing metric.
  If confidence < 90: downgrade block to warning (post comment but don't fail check).
  If confidence < 70: skip the rule entirely (auto-disabled per data-confidence.md).
```

## Test cases

```text
Integration test:
  - PR with "[x] No AI assistance used" -> PASS.
  - PR with "[x] AI used for code suggestions" + "[x] High" -> PASS.
  - PR with "[x] AI used for code suggestions" but no confidence -> FAIL.
  - PR with no AI metadata at all -> FAIL.
  - PR with "emergency-fix" label -> PASS (override logged).
  - PR with low confidence metric -> warning instead of block.
  - PR with very low confidence metric (< 70) -> rule skipped entirely.
```
