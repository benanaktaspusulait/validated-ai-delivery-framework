# T4.9 — Recommendations / Playbook UI

**Phase:** 4  
**Owner:** Platform Engineer  
**Estimate:** 2 days  
**Depends on:** T2.8

## Objective

Build the Recommendations and Playbook screen.

## Acceptance criteria

- [ ] Active recommendations by team (from recommendations table).
- [ ] Playbook: problem -> recommended action -> owner -> status.
- [ ] Override trend chart.
- [ ] Dynamic WIP recommendation with reason.

## Technical notes

```text
Screen: /teams/{teamId}/recommendations

Sections:
  1. Active Recommendations:
     Table: Type | Severity | Message | Status | Created
     - split_large_ai_prs | warning | "Large AI PRs..." | open | 2025-02-10
     - reduce_wip | info | "Review debt high..." | open | 2025-02-12
     Actions: Mark as resolved, Dismiss.

  2. Playbook:
     Table: Problem | Severity | Recommended Action | Owner | Status
     - Review debt high | High | Reduce AI WIP by 20% next sprint | Team Lead | open
     - Defect rate rising | Critical | Require senior review for AI PRs | Tech Lead | open
     - Metadata missing | Medium | Enable CI metadata enforcement | Platform | resolved

  3. Override Trend:
     Line chart: emergency overrides per sprint over last 6 sprints.
     Reference line at 3 (retro trigger) and 5 (hard limit).

  4. Dynamic WIP:
     Card showing: current WIP, recommended WIP, reason.
     Trend: arrow showing increase/decrease from last sprint.
```

## Test cases

```text
- Recommendations loaded from database.
- Mark recommendation as resolved -> status updated.
- Override trend chart renders with data.
- Dynamic WIP card shows correct values.
```
