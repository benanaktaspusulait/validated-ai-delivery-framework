# Psychological Safety

Canonical reference for the psychological safety pulse and its gating rules. A baseline is taken in Phase 0 and re-run in Phase 3 and at the quarterly forum.

## Pulse questions

Scored 1 (strongly disagree) to 5 (strongly agree):

```text
I feel safe declaring AI usage in my work.
I feel safe challenging AI-generated output.
I feel safe saying I do not understand AI-generated code.
Our team reviews AI-assisted code carefully.
AI metrics are used to improve the system, not blame individuals.
I do not feel pressured to use AI when it is not helpful.
```

## Gating rules

```text
Phase 0: record the baseline average. If baseline < 3.5, do not start data collection; address culture and trust first.
Phase 3 and beyond: if the average score is below 3.5, pause metric expansion and run a team retro before adding any enforcement.
```

## Why it matters

The pulse is the trust circuit-breaker for the whole programme. AI usage disclosure must never be punished; the cultural rules in `docs/governance-and-privacy.md` exist to keep declaration safe so the metrics stay honest.

## Briefing message

```text
We are not measuring individual productivity. We are validating whether the system can use AI safely.
```
